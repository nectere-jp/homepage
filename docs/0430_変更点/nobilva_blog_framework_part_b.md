## B1. クラスタ全体マップ

ハブ記事4本＋子記事 約20本の構成を目指す。緊急受け皿は独立。

```
時間軸（大）
 └ ハブ：野球と勉強の両立に悩む保護者の方へ — 時間のつくり方
   ├ 子記事5〜7本

進路軸(大・最優先)
 └ ハブ：野球をがんばる中高生の進路の選び方 — スポーツ推薦・指定校・一般
   ├ 子記事5〜7本

自己認識軸（中）
 └ ハブ：野球で得た力を勉強につなげる — 自信と習慣の立て直し
   ├ 子記事3〜5本

関係性軸（小・ロングテール）
 └ ハブ：野球家庭の親子コミュニケーション — 声かけと家庭内の役割分担
   ├ 子記事3〜4本

緊急受け皿（独立）
 └ メガスタ破産（既存・リライト不要）
```

## B2. 記事ステータス定義

各記事のステータスを以下の6つから選ぶ：

| ステータス | 意味 |
|---|---|
| **未着手** | キーワードは決まっているが記事化していない |
| **企画中** | Part C を記入中（見出し・要旨設計） |
| **執筆中** | Claude API で本文生成中／編集中 |
| **公開済み** | 公開・インデックス完了 |
| **リライト待ち** | 公開済みだが A11 のリライト基準に該当 |
| **アーカイブ** | 削除・統合・終了判断済み |

## B3. リライト・新規執筆の優先順位ルール

毎月の作業着手順は以下：

1. **GSC 流入記事のリライト最優先**
   - 「スポーツ推薦 対策」「スポーツ推薦 英検」「スポーツ推薦対策 いつから」
   - 既に流入があり、野球特化に書き直すだけで CV 期待値が上がる

2. **進路軸ハブ記事の新規執筆**
   - 進路軸は最優先クラスタ
   - ハブができれば子記事の内部リンク先になる

3. **時間軸ハブ記事の新規執筆**
   - 進路軸の次に流入期待が大きい

4. **進路軸・時間軸の主要子記事**
   - 既存キーワードで野球特化リライト
   - 追加採用キーワードで新規執筆

5. **自己認識軸ハブ＋子記事**

6. **関係性軸ハブ＋子記事**
   - ロングテール狙い、CV 期待は深く狭い

7. **季節性のある記事**
   - A11 の年間カレンダー通り、シーズン60日前にリライト着手

## B12. ボード更新ルール

このドキュメントは **Claude Code と一緒に更新**する。

**更新タイミング**
- 記事のステータス変更時（即時）
- 月次レビュー時（毎月1回・必須）

- 半年計画見直し時（半年に1回）

- Claude Code で CMS の状態と Part B を同期
- ステータスが食い違っていた場合、CMS を正とする
