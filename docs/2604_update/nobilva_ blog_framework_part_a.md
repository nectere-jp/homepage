## A1. ブログの位置づけ

### 役割

Nobilva ブログは、**Google 検索および LLM 経由の情報探索行動から流入を獲得し、無料学習診断申し込みへ導くチャネル**。Instagram とは独立した流入経路として運用する。

### KPI

最重要指標は**無料学習診断申し込み数**。PV や検索順位は中間指標であって最終目標ではない。

### Instagram との関係：独立チャネル

Instagram とブログは**独立した流入チャネル**として運用する。両者を無理に連動させず、それぞれが独立に成果を出す設計にする。

|          | Instagram          | ブログ                           |
| -------- | ------------------ | -------------------------------- |
| 入口     | フィード偶発接触   | Google検索／LLM経由の情報探索    |
| 主読者   | 保護者特化         | 保護者＋生徒（記事ごと使い分け） |
| 文体     | 感情の言語化       | 情報提供＋共感＋根拠             |
| 機能     | 共感・ブランド認知 | 課題解決・信頼構築・CV           |
| 完結性   | 単発で完結         | クラスタ内で相互参照             |
| 接触深度 | 浅く広く           | 深く局所的                       |

役割が独立しているため、Instagram と完全に重複する感情共感だけの記事は書かない。ブログは検索意図に直接応える「情報を持って帰れる場所」として設計する。

### LLM 対応（AEO）の意識

ChatGPT・Gemini・Perplexity などの LLM が情報源として引用する可能性を意識する。LLM は構造化され信頼できる情報源を引用する傾向があるため、以下を徹底する：

- **質問形式の見出し**（「スポーツ推薦の英検要件はいつから準備すべきか」など）
- **冒頭で結論を先に提示**（リード文に答えを含める）
- **構造化された記述**（FAQ・箇条書き・定義の明示）
- **一次情報の出典明記**（公的機関・学校公式・統計データ）
- **数字・固有名詞の正確性**（曖昧な表現を避ける）

これは Google SEO とも矛盾せず、むしろ E-E-A-T（経験・専門性・権威・信頼）に整合する。

### やらないこと

- Instagram と完全に重複する感情共感だけの記事
- ビッグワード正面突破（「家庭教師」「中学生 勉強」など、競合が強すぎる）
- ボリューム狙いの量産記事（記事品質と内部リンクの密度を優先）
- 出典のない数字・統計の引用

## A2. 想定読者プロフィール

### 保護者読者像（メイン）

野球をがんばる中高生（中学硬式クラブチーム／中学軟式部活／高校野球部）の母親、30代後半〜40代。送迎・洗濯・補食準備で時間を削られ、子の野球は応援したいが「勉強が後回し」「内申点が下がる」「スポーツ推薦も一般進学も選択肢を残したい」という不安を抱えている。父親が野球経験者・母親が勉強担当という分担の家庭が典型。

**検索行動の特徴**

- 夜、子が寝た後にスマホで検索
- 答えだけではなく**「同じ立場の人がどうしているか」**を知りたい
- 業者の宣伝臭に敏感、押し売りトーンを嫌う
- 競合記事を3〜5本読み比べる

Instagram のメインターゲットと共通だが、ブログでは情報精度がより求められる。

### 生徒読者像（サブ・絞り込み）

**強豪校・準強豪校でレギュラーまたはレギュラー候補の中学2年〜高校2年**。野球を高校・大学まで続けたいが、勉強時間が確保できず学校の成績が下がっている。スポーツ推薦も一般進学も両方の選択肢を残したいという課題感を持つ。

**この層に絞る理由**

- Nobilva のサービス（日割り学習計画・週1面談・進捗確認）が解決できる課題と一致
- 「両立したいけど方法がわからない」という具体的悩みが明確
- 進路選択を真剣に考え始めている学齢
- 保護者と本人の双方が情報を求めている

**絞り込みから外す層（記事ターゲットから明示的に外す）**

- 弱小校エンジョイ系（両立への課題感が薄い）
- 強豪校で完全に推薦狙いの上位層（一般進学を考えていない）
- 部活引退済みで野球と決別している層（引退後系記事は別途扱うが、野球文脈は薄める）

**検索行動の特徴**

- 通学電車・寝る前・遠征バスの中で検索
- **具体的な行動**を求める（精神論は読まない）
- 体験談・他人の例を見て「自分でもできそう」と思いたい
- 親に見せられる内容かどうかも気にしている

### 副次的な読者

- 父親（野球経験者）：稀に検索するが、メインターゲットにはしない
- 中学受験を控えた小6保護者：リトル・シニア所属でこの先を見据えて検索（進路軸記事で拾える）
- 指導者（監督・コーチ）：ターゲット外、書き分けは不要

## A3. 主読者別トーン方針

記事ごとに**主読者を1つ決めて書く**（揺らさない）。「保護者向け」「生徒向け」「両方向け」の3パターン。

### 保護者向け記事

**構成パターン**：共感 → 状況整理 → 情報提供 → 提案 → CTA

**文体ルール**

- 二人称：「保護者の方」「ご家族」
- お子さんの呼称：「お子さん」（「息子さん」「お子様」は使わない）
- 専門用語は必ず注釈（推薦の種類、評定の仕組みなど）
- 数字・制度・スケジュール感などの**情報精度**を重視
- 結論を先に出す（保護者は時間がない）
- 競合サービスとの違いを必要に応じて明示（業者比較で来た読者向け）

**やらないこと**

- 「このままで大丈夫ですか？」の煽り
- 「手遅れになる前に」の脅し
- 父親・夫を悪役にする構図
- 子どもを責める論調

**CTA テンプレート（保護者向け）**

```
Nobilva は、野球をがんばる中高生のための学習管理サービスです。

・週ごとの日割り学習計画

判断材料を持ち帰っていただく場として、ご利用ください。

ご入会後30日以内であれば全額返金保証もございますので、
まずはお気軽にご相談ください。

🔗 www.nectere.jp/ja/services/nobilva
```

### 生徒向け記事

**構成パターン**：共感 → 自分ごと化 → 具体アクション → 成功イメージ → 親に相談する一押し

**文体ルール**

- 二人称：「あなた」「自分」
- フランクすぎない（友達口調は信頼を損なう）
- 具体的アクション（時間・回数・順番）を入れる
- 体験談やシミュレーションを混ぜる
- 「でも、できなかったらどうしよう」への先回りフォロー
- 親に共有しやすい内容を意識する

**やらないこと**

- 上から目線（「〜しなければいけません」）
- 説教調(「努力が足りない」)
- 親を悪者にする構図
- 過度に甘やかす表現（「無理しなくていい」を多用しない）

**CTA テンプレート（生徒向け）**

```
Nobilva は、野球をがんばる中高生のための学習管理サービスです。
週ごとの学習計画を一緒に作って、週1回のオンライン面談で進捗を確認していきます。

「自分で計画を立てても続かない」「何から手をつけたらいいかわからない」
と感じているなら、まずは無料学習診断でメンターと話してみてください。
今の練習スケジュールと得意・苦手を聞いたうえで、続けられそうな学習プランを
一緒に考えます。

お父さん・お母さんと一緒に話を聞いてもらうのも、一つの方法です。
30日以内であれば全額返金保証付きなので、家族で相談する材料として
使ってもらえれば。

🔗 www.nectere.jp/ja/services/nobilva
```

### 両方向け記事

**構成パターン**：リードで「保護者の方も、本人の方も」と明示 → 各章でどちらに語りかけているかを明確化 → 結論はどちらにもアクション可能なものに

**文体ルール**

- 章ごとに語り手の宛先を切り替える（混ぜない）
- リード文と末尾は両方に届く言葉で
- 章タイトルで宛先を示す（「保護者の方へ」「中高生の皆さんへ」）

**使う場面**：「野球 勉強」「部活引退後 勉強」など、検索者像が分かれるキーワードのみ。基本は保護者か生徒のどちらかに寄せる方が記事として強い。

**CTA テンプレート（両方向け）**


## A4. 用語の整理

Instagram フレームワークと**完全統一**する。ブログ独自の用語は作らない。

### 避けるべき表現

- ❌ 少年野球（学童野球＝小学生限定の意味）
- ❌ 球児（甲子園イメージで中学生に違和感）
- ❌ スポ少（学童期の俗称）
- ❌ 少年（古風）

### 公式に使う表現

- ✅ 野球をがんばる中高生
- ✅ 中学硬式野球（クラブチーム）／中学軟式野球（部活）／高校野球部
- ✅ リトルシニア・ボーイズリーグ・ポニーリーグ・ヤングリーグ
- ✅ 中学生選手・高校生選手・野球部員
- ✅ 強豪校・野球強豪校

### 進路関連の用語

- ✅ スポーツ推薦／指定校推薦／公募推薦／総合型選抜（一般入試との対比で書くとき）
- ✅ 評定平均／内申点（「内申」は口語、本文では避ける）
- ✅ 文武両道（使うが、安易に乱発しない）
- ❌ 「スポ推」（業界俗語）

### サービス言及の用語

- ✅ Nobilva は「**野球をがんばる中高生のための学習管理サービス**」
- ✅ サービス三本柱：「日割り学習計画／週1回オンライン面談／毎日の進捗確認」（変更しない）
- ✅ 「30日全額返金保証」「月20名限定 無料学習診断」（ピボット後のオファーを正確に表記）
- ❌ AI 分析・親ダッシュボード・月次レポートなど、実態のない機能を書かない

## A5. 4軸クラスタ構造とハブ記事の役割

### 全体像

```
時間軸（大）
 └ ハブ記事：「野球と勉強の両立に悩む保護者の方へ — 時間のつくり方」
   └ 子記事5〜7本

進路軸（大・最優先）
 └ ハブ記事：「野球をがんばる中高生の進路の選び方 — スポーツ推薦・一般・指定校」
   └ 子記事5〜7本

自己認識軸（中）
 └ ハブ記事：「野球で得た力を勉強につなげる — 自信と習慣の立て直し」
   └ 子記事3〜5本

関係性軸（小・ロングテール）
 └ ハブ記事：「野球家庭の親子コミュニケーション — 声かけと家庭内の役割分担」
   └ 子記事3〜4本

緊急受け皿（独立）
 └ メガスタ破産記事（既存・触らない）
```

### ハブ記事の役割

ハブ記事は**クラスタの入口・地図・着地点**を兼ねる。役割は3つ：

1. **検索流入の入口**：ビッグワード〜ミドルワードで検索した読者を最初に受け止める
2. **クラスタの目次**：その軸で扱うトピック全体を俯瞰し、子記事へ送る
3. **CV 動線の起点**：軸全体を読んだ読者が無料学習診断に進む最終地点

### ハブ記事の特徴

- **重量ボリューム**（6,000〜8,000字）
- 各章で軸内のサブトピックを概観し、詳細は子記事にリンク
- 子記事から見ても「親記事」として参照される
- 記事最後に**軸全体としての CTA**を置く（個別子記事の CTA より上位）

### 子記事の役割

子記事は**特定の検索意図に答える専門記事**。役割は2つ：

1. **ロングテール流入の獲得**：具体的キーワードで検索した読者を直接捕まえる
2. **ハブ記事との相互参照で滞在時間を伸ばす**：関連子記事・ハブ記事への内部リンクで回遊を促進

### 子記事の特徴

- **標準ボリューム**（3,500〜5,000字）または **軽量ボリューム**（2,000〜3,000字、関係性軸など）
- 1記事で1つの検索意図に答えきる
- 記事末尾でハブ記事と関連子記事に2〜3本リンク
- CTA は記事の文脈に沿った具体的な誘導

### 緊急受け皿カテゴリ

メガスタ破産記事は4軸クラスタの外に置く。

- 既存記事のリライトはしない
- 4軸クラスタのハブ記事から**リンクしない**（流入を分離）
- ただし、サイト全体のサイドバー／フッターからは到達可能にしておく
- 別の塾・予備校が破産した場合、同型記事を増やす可能性あり（受け皿カテゴリとして温存）

## A6. 記事構成テンプレート（マズロー組み込み）

### 二段構造の考え方

ブログ記事は**2つのフレームを重ねて設計**する：

- **第1層（クラスタ軸）**：どの悩みカテゴリの記事か（時間／進路／関係性／自己認識）
- **第2層（マズロー）**：記事内をどう展開するか（章立てフレーム）

クラスタ軸はキーワードを4分類するための装置、マズローは記事内の感情・思考の流れを設計するための装置。

### マズロー5段階を章立てフレームに翻訳

野球家庭文脈に翻訳した5段階：

| マズロー段階   | 野球家庭文脈での意味               | 記事内の役割                       |
| -------------- | ---------------------------------- | ---------------------------------- |
| **生理的欲求** | 物理的な余裕（時間・体力・睡眠）   | 「現実問題として何が起きているか」 |
| **安全欲求**   | 不安の解消（進路・将来・選択肢）   | 「何が不安なのかを言語化」         |
| **所属欲求**   | 仲間・同じ立場の家庭の存在         | 「同じ悩みを抱える人たちがいる」   |
| **承認欲求**   | 状況改善・成果・周囲からの評価     | 「正しく取り組めば道は開ける」     |
| **自己実現**   | 子ども自身が選択肢を持つ・自走する | 「最終的にどうなりたいか」         |

### 標準記事構成テンプレート（5章＋リード＋CTA）

```
【リード】（200〜300字）
- 検索者の状況を描写
- 結論を先に出す（LLM 引用対応）
- 記事で得られることを明示

【1章：生理的欲求 — 現実の物理的問題を描写】（400〜600字）
- 何が起きているか（時間がない／成績が下がっている／会話が減った 等）
- 統計や一次情報があれば引用
- 読者の状況を客観的に整理

【2章：安全欲求 — 不安の言語化と整理】（500〜800字）
- 何が不安なのか（進路・将来・選択肢の喪失）
- 不安の正体を分解
- Instagram と違って情報提供で不安を解消する

【3章：所属欲求 — 同じ立場の家庭の存在】（500〜800字）
- 「あなただけではない」を提示
- 一般的な傾向・データ
- ケーススタディ（仮名でも可）
- 競合記事には少ない要素、差別化ポイント

【4章：承認欲求 — 解決の方向性と具体策】（800〜1,200字）
- 状況改善のための具体アクション
- 実行可能なステップ
- 子記事への内部リンクをここで配置

【5章：自己実現 — 最終的なゴール像】（400〜600字）
- お子さん（または本人）が最終的にどうなれるか
- 短期ゴールではなく長期ビジョン
- ここで Nobilva が果たす役割を自然に位置づける

【CTA】（200〜300字）
- A3 のテンプレートを使用
- 主読者に応じて使い分け
```

合計：3,500〜5,000字の標準構成。これがブログ記事の**デフォルト型**。

### ボリューム別の章立て調整

**ハブ記事（重量・6,000〜8,000字）**

5章構成は維持しつつ、各章を1.5〜2倍に拡張。または6〜8章構成に拡張：

```
1章：生理的欲求（拡張）
2章：安全欲求（拡張）
3章：所属欲求（拡張）+ 章末でクラスタ全体の地図を提示
4章：承認欲求（拡張）+ 子記事へのリンクを複数配置
5章：自己実現（拡張）
6章（追加）：よくある質問（FAQ 形式・LLM 引用対応）
7章（追加）：関連トピックへのナビゲーション
```

**軽量記事（2,000〜3,000字・関係性軸など）**

5章構成を3章に圧縮：

```
1章：現実問題の描写（生理的＋安全欲求を合体）
2章：同じ立場・解決の方向性（所属＋承認欲求を合体）
3章：ゴール像と Nobilva の役割（自己実現）
```

### 章立てフレームの使い方ルール

- **5章すべての要素を必ず含める**（圧縮版でも全要素をカバー）
- **章タイトルにマズロー段階名を出さない**（読者には見せない・裏側のフレーム）
- **章タイトルは検索意図に答える形で書く**（例：「練習で勉強時間が取れない時に、何を最初に変えるか」）
- **各章の冒頭に小見出し的な1文**を置く（読み飛ばし対策）
- **保護者向けと生徒向けで章タイトルの語り口を変える**（A3 のトーン方針に従う）

### LLM 引用を意識した記述ルール

- 各章の冒頭で**結論を1〜2文で示す**
- 数字・固有名詞・制度名を**正確に**書く
- FAQ 章を入れる場合、**質問形式の見出し**を使う
- 出典がある情報は**出典を明記**
- 曖昧な表現（「〜と言われています」）を避け、**主語を明確に**する

## A7. 内部リンク戦略

### 目的

- 滞在時間を伸ばす（SEO 評価向上）
- クラスタ内の回遊で信頼を蓄積
- CV ページ（無料学習診断・LP）への到達率を上げる

### リンク構造の3層

```
第1層：ハブ記事 ⇔ 子記事（軸内・双方向）
第2層：子記事 ⇔ 子記事（同軸内・関連性ベース）
第3層：全記事 → CV ページ（無料学習診断・LP）
```

**第1層**：子記事は冒頭近くで「この記事は『野球と勉強の両立』クラスタの一部です」とハブを示す。ハブ記事は4章（承認欲求＝具体策）で各子記事へリンクする。

**第2層**：同じ軸内の関連子記事に2〜3本リンク。記事末尾の「あわせて読みたい」セクションに配置。

**第3層**：全記事末尾の CTA から無料学習診断ページ／LP へ。記事中盤にも自然なタイミングで1本入れてもいい（やりすぎ注意）。

### 軸をまたぐリンクのルール

クラスタ間のリンクは**近接性に応じて緩急をつける**。

**近接軸（積極的にリンクしてよい）**

- 関係性軸 ⇔ 自己認識軸（家庭内の関係性と本人の自己認識は連続的）
- 時間軸 ⇔ 進路軸（時間の使い方は進路選択と直結）

**遠隔軸（リンクは絞る）**

- 時間軸 ⇔ 関係性軸（テーマが離れすぎている）
- 進路軸 ⇔ 自己認識軸（情報系と感情系で読者の状態が異なる）

**例外**

- 「野球 勉強」のような複数軸にまたがるキーワードのハブ記事は、複数軸へのナビゲーションが自然
- 記事内容の文脈に応じて、遠隔軸でも具体的に必要なリンクは置いてよい（盲目的に禁止しない）

### CV ページへの導線設計

最終目的地は**無料学習診断申し込み**。ここまでの導線を3パターンで設計：

**パターン1：記事内 CTA から直接**

- 全記事末尾のテキスト CTA
- 記事中盤のバナー（任意）

**パターン2：Nobilva LP 経由**

- サービスをまだ知らない読者向け
- 記事末尾の「Nobilva とは」リンクから LP へ送る
- LP で詳細を理解させてから無料診断に誘導

**パターン3：直接無料診断ページ**

- すでに Nobilva を認知している読者向け
- 記事末尾に「→ 無料学習診断はこちら」を明示

判断基準：**初出記事（流入の入口になりやすい記事）はパターン2、深いキーワードの子記事はパターン3**。

### アンカーテキストのルール

- 「こちら」「詳しくはこちら」を使わない（SEO 上も読者体験上も悪手）
- 内部リンクは**記事タイトルそのもの**または**キーワードを含む自然な文**で
- 例：✅「**スポーツ推薦の英検要件はいつから準備すべきか**」← 記事タイトル
- 例：✅「**野球と勉強の両立に悩む保護者の方へ**でも詳しく扱っています」
- 例：❌「詳しくは**こちら**」

### サイト構造への要件

- パンくずリストを設置（クラスタ階層を可視化）
- カテゴリページに4軸を明示（時間／進路／自己認識／関係性）
- サイドバー or フッターに緊急受け皿カテゴリを別建て
- 各記事に「同じ軸の他の記事」セクション（自動生成または手動キュレーション）

## A8. Nanobanana ブログ用共通プロンプト

### 基本方針

Instagram フレームワークの A6 共通プロンプトを土台に、ブログ用にチューニング。**Nectere の標準画像生成ツールとして Nanobanana を統一運用**。Unsplash は新規利用停止、既存記事のリライト時に差し替え。

### Instagram 版との違い

| 項目         | Instagram 版             | ブログ版                 |
| ------------ | ------------------------ | ------------------------ |
| アスペクト比 | 4:5（縦・1080×1350）     | 16:9（横・1920×1080）    |
| 用途         | カルーセル単体スライド   | 記事ヘッダー＋本文中     |
| トーン       | 一貫した感情演出         | 記事カテゴリで微調整     |
| 顔の扱い     | 顔出ししない             | 顔出ししない（共通）     |
| テキスト前提 | あり（オーバーレイ前提） | なし（純粋ビジュアル）   |
| 彩度         | 抑えめ                   | やや明るめ（読み物向き） |

### 共通プロンプトテンプレート（Gemini チャット用・1ブロック）

シーン固有の描写を最初に書き、その後にこの共通ブロックを続ける形で1メッセージとして送信する。

```
[ここにシーン固有の描写を1〜3行で記述]

Cinematic documentary-style photograph, Japanese household or school setting,
natural lighting with warm color grading, soft film grain,
slightly desaturated colors with warm undertones but readable brightness,
no faces visible (composition shows only objects, hands, back views,
or silhouettes), shallow depth of field, 35mm film aesthetic,
quiet and contemplative mood, horizontal 16:9 aspect ratio (1920x1080),
optimized for blog article header, no text, no logos, no watermarks.

Avoid: faces, people looking at camera, smiling people,
American baseball uniforms or MLB style, oversaturated colors,
harsh lighting, stock photo aesthetic, cartoon or illustration style,
text overlays, watermarks, logos, posters, signs with letters.
```

### Instagram 版との主な変更点

- `vertical 4:5 aspect ratio (1080x1350)` → `horizontal 16:9 aspect ratio (1920x1080)`
- `optimized for mobile Instagram feed` → `optimized for blog article header`
- `muted desaturated colors` → `slightly desaturated colors with warm undertones but readable brightness`
- `text, watermarks, or logos` → `text overlays, watermarks, logos, posters, signs with letters`（テキスト混入をより明示的に排除）

### カテゴリ別のシーン描写パターン

ブログ4軸ごとに、シーン描写の方向性をテンプレ化しておく。

#### 時間軸の記事

**主題**：時間が足りない／練習で疲れている／スキマ時間／早朝・深夜

**シーン例**

```
- An open notebook on a wooden desk with morning sunlight streaming in,
  a baseball cap placed beside it, suggesting early morning study time,
  Japanese student room, no person visible.

- A student bag and baseball bag on the floor of a Japanese entryway,
  late evening light, suggesting return from practice, focus on objects.

- A train window seat at dusk, a textbook open on the lap with hands
  visible holding a pencil, blurred suburban scenery outside,
  commuting study scene, no face visible.
```

#### 進路軸の記事

**主題**：志望校選び／進路相談／推薦・一般／面接対策／願書

**シーン例**

```
- A desk with college brochures, a notebook with handwritten notes,
  a calculator and reference books, soft afternoon light through window,
  Japanese household setting, hands visible holding a pen.

- A high school exterior shot at golden hour, viewed from a respectful
  distance, suggesting future possibilities, no people visible,
  Japanese suburban school architecture.

- An open desk calendar with circled dates, study planner with notes,
  a teacup nearby, warm desk lamp light, evening study session,
  no person visible.
```

#### 自己認識軸の記事

**主題**：自信喪失／挫折感／立て直し／苦手克服／継続できない自分への自己嫌悪

**シーン例**

```
- A worn study notebook left open with handwritten notes,
  some pages crumpled, soft window light, melancholic stillness,
  no person visible, Japanese student room.

- A student's hand holding a pencil over an open textbook,
  shoulders slightly slumped (back view only), warm desk lamp light,
  contemplative mood, evening setting.

- A baseball glove and a closed textbook side by side on a desk,
  symbolic composition showing the dual identity, soft morning light,
  no person visible.
```

#### 関係性軸の記事

**主題**：親子の声かけ／夫婦の温度差／家族での会話／父母会／兄弟関係

**シーン例**

```
- Two cups of tea on a kitchen table, one half-empty, suggesting
  a recent conversation, warm evening light, Japanese household,
  no people visible.

- A mother's hand placing a folded sports towel and a snack box
  near a baseball bag in an entryway, only hands and objects visible,
  warm morning light.

- A family dining room at evening with empty plates and a baseball cap
  on a chair, suggesting a recently finished family dinner,
  warm light, no people visible.
```

### 生成・選定のルール

- 1記事あたり3〜5枚生成して選定
- カテゴリ別シーン描写パターンをベースに、記事固有の状況描写を加える
- 「日本の家庭」「日本の学生」を必ず含める（西洋風になりがち）
- 野球用品はメジャーリーグ感が出ないか確認（NPB / 日本の高校野球風に）
- 記事内で複数枚使う場合は色温度・粒子感の統一感をチェック

### Instagram 画像との差別化

- Instagram は感情演出のため**やや暗め・彩度低め**
- ブログは長文を読ませるため**やや明るめ・読みやすさ重視**
- 同じ主題でも、Instagram で使った構図は**ブログで使い回さない**
- ブランドの一貫性は色温度（暖色系）とドキュメンタリー的視点で担保する

## A9. 画像配置パターン

### 配置パターンは3種

記事のボリュームと役割に応じて、3パターンから選ぶ。

#### パターン1：シンプル構成（軽量〜標準記事向け、3枚）

```
[ヘッダー画像]            ← 記事の印象を決める
↓
リード文
↓
1〜2章
↓
[中盤画像]                ← 読みやすさのリズム
↓
3〜4章
↓
[CTA直前画像]             ← CV 直前の温度を上げる
↓
5章 + CTA
```

**向いている記事**

- 関係性軸の子記事
- 自己認識軸の軽量記事
- 軽量〜標準ボリューム（2,000〜5,000字）

#### パターン2：リッチ構成（ハブ記事・標準記事向け、5〜6枚）

```
[ヘッダー画像]
↓
リード文
↓
[1章冒頭画像]
1章
↓
[2章冒頭画像]
2章
↓
[3章冒頭画像]
3章
↓
[4章冒頭画像]
4章
↓
[5章冒頭画像 / CTA直前画像]
5章 + CTA
```

**向いている記事**

- 全ハブ記事
- 進路軸・時間軸の標準記事
- 重量ボリューム（5,000〜8,000字）

#### パターン3：最小構成（緊急・ノウハウ記事向け、1〜2枚）

```
[ヘッダー画像]
↓
記事本文（章間画像なし）
↓
[CTA直前画像（任意）]
↓
CTA
```

**向いている記事**

- 即応性が求められる記事
- ノウハウ系で本文の流れを切りたくない記事
- 軽量ボリューム（2,000〜3,000字）

### 配置パターンの選び方

| 記事タイプ             | 推奨パターン          |
| ---------------------- | --------------------- |
| ハブ記事（4本）        | パターン2（リッチ）   |
| 進路軸の主要子記事     | パターン2（リッチ）   |
| 時間軸の主要子記事     | パターン1または2      |
| 自己認識軸の子記事     | パターン1（シンプル） |
| 関係性軸の子記事       | パターン1（シンプル） |
| ロングテール・ノウハウ | パターン3（最小）     |

### 画像のサイズ・形式

- **解像度**：1920×1080（16:9）
- **ファイル形式**：WebP（容量最適化）／JPEG（互換性優先）
- **ファイル名規則**：`記事スラッグ_位置.webp`（例：`sports-recommendation-eiken_header.webp`）
- **alt テキスト**：シーン描写を簡潔に書く（SEO・アクセシビリティ対応）
  - 例：「夕暮れの机に置かれた野球グローブと教科書」
  - 「画像」「写真」だけの alt は避ける

### 画像と本文の関係

- 画像は本文の補強であって、本文で語られていない情報を画像だけに載せない
- 章冒頭画像は、その章の主題を**比喩的に**表現する（直接的な図解ではない）
- 同じ記事内で同じ被写体（例：野球バッグ）を繰り返さない
- 写真の中に文字（看板・本のタイトル等）が映り込んでいないか確認

### 既存記事のリライト時の差し替えルール

- Unsplash 画像は段階的に差し替え
- リライト対象記事の優先順位（A5 のハブ記事優先＋GSC 流入記事優先）に沿って差し替え
- 一気に全記事差し替える必要はない（運用負荷を考慮）

## A10. メタディスクリプション・OGP・タイトルのルール

### タイトル（title タグ）

**目的**：検索結果での CTR を最大化する／記事内容を正確に伝える／ブランド認知を作る

**標準フォーマット**

```
[検索意図に答える要素] | Nobilva
```

**文字数の目安**：30〜38字（Google検索結果での表示崩れを避ける）

**作成ルール**

- メインキーワードは**前半に配置**
- 数字・固有名詞を入れて具体性を上げる（「3つの方法」「中3保護者向け」など）
- 煽り（「絶対」「最強」「これだけ」）は使わない
- 同じクラスタ内のタイトル形式を揃えすぎない（検索結果で並んだ時に冗長）
- ブランド名は末尾に固定（`| Nobilva`）

**主読者別のタイトル例**

| 主読者     | キーワード例               | タイトル例                                                                |
| ---------- | -------------------------- | ------------------------------------------------------------------------- |
| 保護者向け | スポーツ推薦 英検          | スポーツ推薦で英検は必要？野球部保護者が知っておきたい準備時期 \| Nobilva |
| 生徒向け   | 野球部 勉強 ついていけない | 野球部で勉強についていけない時、最初に変えるべき1つのこと \| Nobilva      |
| 両方向け   | 野球 勉強 両立             | 野球と勉強を両立する現実的な方法 — 中高生と保護者向け \| Nobilva          |

**ハブ記事のタイトル**

- ハブ記事は**包括性**を示すタイトルに
- 例：「野球をがんばる中高生の進路の選び方 — スポーツ推薦・指定校・一般入試 \| Nobilva」

### メタディスクリプション

**目的**：検索結果でのクリック判断材料／LLM が記事を要約する際のヒント

**文字数の目安**：120〜140字（Google検索結果での表示崩れを避ける）

**標準フォーマット**

```
[読者の状況描写]。[記事で得られること]。[サービス言及（控えめ）]。
```

**作成ルール**

- 1文目で**検索者の状況**に触れる（「〜で悩む方へ」など）
- 2文目で**記事の中身**を要約（数字・固有要素を含める）
- 3文目で**サービスを軽く触れる**（押し売りトーンを避ける）
- 結論を含める（LLM 引用対応）
- キーワードを自然に含める（詰め込まない）

**例**

```
スポーツ推薦の英検要件はいつから準備すべきか、野球部保護者向けに、
中3夏までの準備計画を3段階で整理しました。志望校別の英検級・スコア基準も
解説。Nobilva では野球をがんばる中高生の学習管理を行っています。
```

### OGP（Open Graph Protocol）

**目的**：SNS シェア時の表示最適化／LINE やメッセージで共有された時の見た目

**設定項目**

| 項目             | ルール                                                        |
| ---------------- | ------------------------------------------------------------- |
| `og:title`       | title タグと同一(または若干短縮）                             |
| `og:description` | メタディスクリプションと同一                                  |
| `og:image`       | 記事ヘッダー画像（1920×1080）／専用 OGP 画像があれば 1200×630 |
| `og:type`        | `article`                                                     |
| `og:site_name`   | `Nobilva`                                                     |
| `og:url`         | 記事の正規 URL                                                |

**OGP 画像の補足**

- 当面は記事ヘッダー画像（1920×1080）をそのまま流用
- LINE・Twitter での切り取られ方を確認（中央が切れないか）
- 将来的に OGP 専用画像（1200×630・テキスト入り）を作る選択肢は残すが、A9 の方針通り**当面はテキストオーバーレイなしで運用**

### URL（スラッグ）

**ルール**

- 英数字＋ハイフン
- ローマ字ではなく**英訳**を優先（LLM・グローバル検索を意識）
- キーワードを含める
- 短くまとめる（30文字以内目安）

**例**

| キーワード            | スラッグ                                   |
| --------------------- | ------------------------------------------ |
| スポーツ推薦 英検     | `sports-recommendation-eiken-requirements` |
| 野球 勉強 両立        | `baseball-study-balance`                   |
| 部活引退後 勉強       | `after-club-retirement-study`              |
| 野球部 ついていけない | `baseball-club-struggling-with-study`      |

### 構造化データ（schema.org）

LLM 引用と Google リッチリザルト対応のため、以下の構造化データを各記事に埋め込む：

- `Article` スキーマ（必須）
  - `headline`、`datePublished`、`dateModified`、`author`、`publisher`、`image`
- `FAQPage` スキーマ（FAQ 章を含む記事のみ）
  - 質問と回答を機械可読な形で
- `BreadcrumbList` スキーマ（推奨）
  - パンくずリスト（カテゴリ階層を機械可読に）

**運用上の注意**

- Claude Code で記事生成時に構造化データも一緒に生成する
- 公開前に Google Rich Results Test で検証

### h1〜h3 の使い方

- `h1`：記事タイトル（ページ内に1つだけ）
- `h2`：章タイトル（マズロー5段階に対応する5章＋CTA章）
- `h3`：章内のサブ見出し（必要な場合のみ）
- `h4` 以下は基本使わない（読みづらくなる）

**章タイトルのルール（再掲）**

- マズロー段階名を**読者には見せない**
- 検索意図に答える形で書く
- キーワードを自然に含める

## A11. 公開後の運用

### 公開直後（公開〜2週間）

- **インデックス確認**：Google Search Console で URL 検査ツールを使い、インデックス申請
- **構造化データ確認**：Rich Results Test でエラーがないか
- **内部リンク確認**：ハブ記事から子記事へ、子記事から関連記事への双方向リンクが繋がっているか
- **CV 動線確認**：CTA から無料学習診断ページへの遷移が正しいか
- **モバイル表示確認**：画像のはみ出し・テキスト崩れがないか

### 月次レビュー（毎月実施）

毎月1回、以下の指標を確認：

| 指標                | 確認ツール      | 目安                                |
| ------------------- | --------------- | ----------------------------------- |
| 記事別 PV           | GA4             | クラスタ別合計を比較                |
| 記事別 CTR          | GSC             | 1.0% 未満は要改善                   |
| 記事別 平均掲載順位 | GSC             | 上位50位以内に入っているか          |
| 流入クエリ          | GSC             | 想定外の検索意図で来ていないか      |
| 直帰率              | GA4             | 80% 超は要改善                      |
| 滞在時間            | GA4             | 2分未満はリード文・冒頭画像の見直し |
| CV 数               | GA4（イベント） | **最重要**                          |
| CV 経由記事         | GA4（経路分析） | どの記事が CV に貢献しているか      |

### 月次レビュー用ダッシュボードツール（要構築）

8項目の指標を毎月手動で集計するのは負荷が高いため、**ダッシュボードツールを内製する**。Claude Code で構築する想定。

**要件**

- GA4 API・Search Console API・自社 DB（CV データ）を統合
- 記事ごとに上記8指標を一覧表示
- クラスタ別（時間／進路／自己認識／関係性）の集計ビュー
- リライト判断基準（後述）に該当する記事を自動ハイライト
- 月次サマリの自動生成

**ピボット計画書のアナリティクスツールとの統合**

- ピボット計画書 §9 のアナリティクスツール（テレアポ・LP・LINE 一元管理）と同じスタックで構築
- ブログ KPI もこのツールに統合する想定（個別ツール乱立を避ける）

**実装優先度**

- フェーズ1：手動入力フォーム＋ Supabase で月次データ蓄積
- フェーズ2：GA4 / Search Console API 連携で自動取得
- フェーズ3：リライト判断の自動アラート

### リライト判断基準

以下のいずれかに該当したらリライトを検討：

1. **掲載順位30〜50位**で停滞して3ヶ月以上動かない
   → タイトル・h2・リード文の見直し
2. **CTR 1.0% 未満**
   → タイトル・メタディスクリプションの見直し
3. **滞在時間2分未満**
   → 冒頭の魅力・画像配置・章構成の見直し
4. **検索クエリと記事内容のズレ**
   → 検索者の意図に合わせて加筆・章追加
5. **競合記事のアップデート**で順位を抜かれた
   → 情報の鮮度・網羅性を上げる

### リライト優先順位

複数の記事で改善が必要な場合の優先順位：

1. **GSC でクリック数の多い記事**（既に流入があり改善効果が出やすい）
2. **ハブ記事**（クラスタ全体への影響が大きい）
3. **進路軸の記事**（CV 期待値が高い）
4. **掲載順位11〜20位の記事**（あと一押しで1ページ目に入る）

### 季節性のある記事のカレンダー管理

特定時期に検索が集中する記事は、**シーズン前にリライト＋再共有**する。年間カレンダーで管理：

| 時期                | 該当記事テーマ                                      | アクション                           |
| ------------------- | --------------------------------------------------- | ------------------------------------ |
| 4月上旬             | ゴールデンウィーク 勉強両立                         | 日付更新・リード文刷新・最新情報追記 |
| 7月上旬             | 部活合宿中 勉強                                     | 同上                                 |
| 7月下旬             | 部活引退後 勉強（夏引退組）                         | 同上                                 |
| 10月下旬            | 部活引退後 勉強（秋引退組）／野球 オフシーズン 勉強 | 同上                                 |
| 1月中旬             | 新学期前の習慣立て直し                              | 同上                                 |
| 春休み前（3月初旬） | 春の学習計画づくり                                  | 同上                                 |

**カレンダー管理の仕組み**

- 月次レビューダッシュボードに季節記事のリマインダー機能を組み込む
- 該当時期の60日前にアラート通知（リライト準備期間を確保）
- 該当時期の30日前にリライト完了目標
- 公開直後の動きを GSC で追跡

「再共有」の具体的内容：日付の更新／冒頭リード文の刷新／関連最新情報の追記／必要に応じてヘッダー画像の差し替え。

### 記事の終了判断

以下の場合は記事を**統合・削除**を検討：

- 公開から1年以上、月間 PV が10未満
- 検索意図が時代遅れになった（制度変更等）
- より良いキーワードを持つ類似記事と統合した方がいい

削除する場合は **301 リダイレクト**で関連記事に流す（リンクジュース保全）。

### Instagram との連動（補足）

A1 で「独立チャネル」と決めたが、運用上のシナジーは活用してよい：

- Instagram で反応の良かった主題は、ブログ記事化を検討
- ブログで PV が多い記事は、Instagram 投稿のテーマに転用
- ただし**完全な内容重複は避ける**（Instagram は感情共感、ブログは情報提供）
