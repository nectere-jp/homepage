#### 8.9.5 Firebase のセキュリティルール

**Firestore Security Rules**
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 認証済みユーザーのみアクセス可能
    match /{document=**} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && 
                      request.auth.token.role == 'admin';
    }
    
    // 公開コンテンツ（一般ユーザーも閲覧可能）
    match /posts/{postId} {
      allow read: if resource.data.status == 'published';
      allow write: if request.auth != null && 
                      request.auth.token.role == 'admin';
    }
    
    match /pressReleases/{pressReleaseId} {
      allow read: if resource.data.status == 'published';
      allow write: if request.auth != null && 
                      request.auth.token.role == 'admin';
    }
    
    match /news/{newsId} {
      allow read: if resource.data.status == 'published';
      allow write: if request.auth != null && 
                      request.auth.token.role == 'admin';
    }
    
    match /members/{memberId} {
      allow read: if resource.data.isPublished == true;
      allow write: if request.auth != null && 
                      request.auth.token.role == 'admin';
    }
  }
}
```

```javascript
rules_version = '2';
```

**Firestore の使用例（`lib/admin/db.ts`）**
```typescript
import { collection, doc, getDoc, getDocs, setDoc, updateDoc, deleteDoc, query, where, orderBy } from 'firebase/firestore';
import { db } from '@/lib/firebase';

export const getPosts = async () => {
  const postsRef = collection(db, 'posts');
  const q = query(postsRef, orderBy('createdAt', 'desc'));
  const snapshot = await getDocs(q);
  return snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() }));
};

export const createPost = async (postData: any) => {
  const postsRef = collection(db, 'posts');
  const newPostRef = doc(postsRef);
  await setDoc(newPostRef, {
    ...postData,
    createdAt: new Date(),
    updatedAt: new Date(),
  });
  return newPostRef.id;
};
```

- **ユーザビリティを重視**
- **人間らしさを表現**（テクノロジーに人間らしさを取り戻す）

### 8.2 カラースキーム

- **ベースカラー**：白、グレー系
- **注意**：ベージュのカラーコードは #ECD9CD（構成案の最初に記載）と #ECD9BD（画像の説明）の2つがありますが、実装時に統一してください
- **カラー使用方針**
  - **ピンク（#FA6B82）**
    - セクションヘッダー（「Service」「Mission」など）

  - **ベージュ（#ECD9CD）**
    - メインコンテンツエリアの背景色
    - 優しく温かみのある印象を与える
    - カード間の余白・セクション間の背景
  - **白**

### 8.8 アニメーション・インタラクション

#### 8.8.1 カードアニメーション

**カードホバーエフェクト**
- **浮き上がり効果**
  - `transform: translateY(-4px)`（ホバー時）
  - `box-shadow` の強化（`0 4px 16px rgba(0,0,0,0.12)`）
  - トランジション：`transition: all 0.3s ease`
- **スケール効果**（オプション）
  - `transform: scale(1.02)`（微細な拡大）
- **カード内要素のアニメーション**
  - CTAボタンの色変化（ピンクの濃淡）
  - 矢印アイコンの右移動（`translateX(4px)`）

**カードのスクロールインアニメーション**
- **フェードイン + スライドアップ**
  - 初期状態：`opacity: 0`、`transform: translateY(30px)`
  - アニメーション後：`opacity: 1`、`transform: translateY(0)`
  - 遅延時間：各カードに0.1sずつ遅延（`animation-delay: 0.1s, 0.2s, 0.3s...`）
  - アニメーション時間：`duration: 0.6s`
  - イージング：`ease-out`

**カードのスタッガーアニメーション**
- サービスカードが順番に表示される
- グリッドレイアウトで左から右、上から下に順次アニメーション

#### 8.8.2 セクションアニメーション

**セクションヘッダーのアニメーション**
- **フェードイン + スライドイン（左から）**
  - 初期状態：`opacity: 0`、`transform: translateX(-20px)`
  - アニメーション後：`opacity: 1`、`transform: translateX(0)`
  - アニメーション時間：`duration: 0.5s`

**セクションタイトルのアニメーション**
- **フェードイン + スライドアップ**
  - 初期状態：`opacity: 0`、`transform: translateY(20px)`
  - アニメーション後：`opacity: 1`、`transform: translateY(0)`
  - アニメーション時間：`duration: 0.7s`
  - セクションヘッダーの0.2s後に開始

#### 8.8.3 CTAボタン・リンクのアニメーション

**CTAボタンのホバーエフェクト**
- **テキストカラーの変化**
  - 通常：`#FA6B82`
  - ホバー：`#E85A6F`（少し濃いピンク）
  - トランジション：`transition: color 0.2s ease`
- **矢印アイコンのアニメーション**
  - 通常位置：`transform: translateX(0)`
  - ホバー時：`transform: translateX(6px)`
  - トランジション：`transition: transform 0.3s ease`
- **背景色の変化**（ボタンスタイルの場合）
  - 通常：透明または白
  - ホバー：ピンクの背景（`#FA6B82`、`opacity: 0.1`）

**CTAボタンのクリックエフェクト**
- **リップルエフェクト**（オプション）
  - クリック位置から円形に広がるアニメーション
  - `duration: 0.4s`

#### 8.8.4 ページ遷移アニメーション

**ページ間のトランジション**
- **フェードトランジション**
  - フェードアウト：`opacity: 1 → 0`（`duration: 0.2s`）
  - フェードイン：`opacity: 0 → 1`（`duration: 0.3s`）
- **スライドトランジション**（オプション）
  - 次のページが右からスライドイン
  - `transform: translateX(100%) → translateX(0)`

#### 8.8.5 スクロールアニメーション

**Intersection Observer API を使用**
- **要素がビューポートに入ったときにアニメーション開始**
- **アニメーションタイプ**
  - フェードイン：`opacity: 0 → 1`
  - スライドアップ：`translateY(30px) → translateY(0)`
  - スライドイン（左）：`translateX(-30px) → translateX(0)`
  - スライドイン（右）：`translateX(30px) → translateX(0)`
  - スケールイン：`scale(0.95) → scale(1)`

**スクロールトリガーの設定**
- **閾値**：要素の20%が表示されたらアニメーション開始
- **一度だけ実行**：アニメーションは1回のみ実行（リセットなし）

#### 8.8.6 ヒーローセクションのアニメーション

**キャッチコピーのアニメーション**
- **タイプライターエフェクト**（オプション）
  - テキストが1文字ずつ表示される
  - `duration: 0.05s per character`
- **フェードイン + スライドアップ**
  - 初期状態：`opacity: 0`、`transform: translateY(40px)`
  - アニメーション後：`opacity: 1`、`transform: translateY(0)`
  - `duration: 1s`、`delay: 0.3s`

**CTAボタンのアニメーション**
- **バウンスイン**（オプション）
  - `animation: bounceIn 0.8s ease-out`
  - `delay: 0.8s`（キャッチコピーの後）

**背景画像/動画のアニメーション**
- **フェードイン**
  - `opacity: 0 → 1`、`duration: 1.5s`
- **ズームイン**（オプション）
  - `transform: scale(1.1) → scale(1)`
  - `duration: 2s`

#### 8.8.7 ナビゲーションのアニメーション

**ヘッダーのスクロール時のアニメーション**
- **固定ヘッダー**
  - スクロールダウン：`transform: translateY(-100%)`
  - スクロールアップ：`transform: translateY(0)`
  - `transition: transform 0.3s ease`

**メニューアイテムのホバーエフェクト**
- **アンダーラインアニメーション**
  - 通常：`width: 0`
  - ホバー：`width: 100%`
  - `transition: width 0.3s ease`
- **カラーの変化**
  - 通常：グレーまたは黒
  - ホバー：ピンク（`#FA6B82`）

**モバイルメニューのアニメーション**
- **スライドイン**（右から左）
  - `transform: translateX(100%) → translateX(0)`
  - `duration: 0.3s`
- **オーバーレイのフェードイン**
  - `opacity: 0 → 0.5`（背景が暗くなる）

#### 8.8.8 ローディングアニメーション

**ページローディング**
- **スケルトンローディング**
  - コンテンツの形状を保持したローディング状態
  - シマーアニメーション（`shimmer effect`）
- **スピナー**（オプション）
  - 軽量なローディングスピナー
  - ピンク（`#FA6B82`）のカラー

**画像のローディング**
- **フェードイン**
  - 画像読み込み完了後に`opacity: 0 → 1`
  - `duration: 0.5s`
- **プレースホルダー**
  - ぼかし効果（`blur`）からクリアに

#### 8.8.9 フォームのアニメーション

**入力フィールドのフォーカスアニメーション**
- **ボーダーの色変化**
  - 通常：グレー
  - フォーカス：ピンク（`#FA6B82`）
  - `transition: border-color 0.2s ease`
- **ラベルのアニメーション**
  - フォーカス時に上に移動
  - `transform: translateY(-20px)`、`scale(0.9)`

**バリデーションエラーのアニメーション**
- **シェイクアニメーション**
  - `animation: shake 0.5s`
  - エラーメッセージのフェードイン

#### 8.8.10 実装推奨ライブラリ

**アニメーションライブラリ**
- **framer-motion**（推奨）
  - React用のアニメーションライブラリ
  - 宣言的なAPI、パフォーマンスが良い
  - 例：`<motion.div animate={{ opacity: 1 }} initial={{ opacity: 0 }}>`
- **react-spring**（代替案）
  - 物理ベースのアニメーション
  - より自然な動き
- **GSAP（GreenSock）**（高度なアニメーションが必要な場合）
  - より複雑なアニメーションに対応

**スクロールアニメーションライブラリ**
- **react-intersection-observer**
  - Intersection Observer APIのReactラッパー
- **AOS (Animate On Scroll)**（シンプルな場合）
  - データ属性で簡単にアニメーション追加

#### 8.8.11 パフォーマンス最適化

**アニメーションの最適化**
- **`will-change`プロパティ**の使用
  - アニメーションする要素に`will-change: transform, opacity`を設定
- **GPUアクセラレーション**
  - `transform`と`opacity`を使用（レイアウト再計算を避ける）
- **`requestAnimationFrame`**の活用
- **アニメーションの無効化**（オプション）
  - ユーザーの設定（`prefers-reduced-motion`）を尊重
  - `@media (prefers-reduced-motion: reduce)`でアニメーションを無効化

**アニメーションのタイミング**
- **デフォルトのタイミング**
  - 短いアニメーション：`0.2s - 0.3s`
  - 中程度のアニメーション：`0.5s - 0.7s`
  - 長いアニメーション：`1s - 1.5s`
- **イージング関数**
  - 標準：`ease`、`ease-out`、`ease-in-out`
  - カスタム：`cubic-bezier(0.4, 0, 0.2, 1)`（Material Design）

#### 8.8.12 アニメーションの例（実装コード例）

**カードのホバーアニメーション（CSS）**
```css
.service-card {
  transition: all 0.3s ease;
}

.service-card:hover {
  transform: translateY(-4px);
  box-shadow: 0 4px 16px rgba(0, 0, 0, 0.12);
}
```

**カードのスクロールインアニメーション（framer-motion）**
```tsx
<motion.div
  initial={{ opacity: 0, y: 30 }}
  whileInView={{ opacity: 1, y: 0 }}
  viewport={{ once: true, amount: 0.2 }}
  transition={{ duration: 0.6, ease: "easeOut" }}
>
  <ServiceCard />
</motion.div>
```

**CTAボタンのアニメーション（framer-motion）**
```tsx
<motion.button
  whileHover={{ scale: 1.05 }}
  whileTap={{ scale: 0.95 }}
>
  <motion.span
```

## 10. 技術要件（一般ユーザー向けページ）

### 9.1 技術スタック
- **フレームワーク**：Next.js
  - App Router（推奨）またはPages Router
  - Server Components / Client Components
  - 静的サイト生成（SSG）とサーバーサイドレンダリング（SSR）の活用
- **スタイリング**
  - Tailwind CSS（推奨）またはCSS Modules、styled-components
- **言語**：TypeScript（推奨）またはJavaScript
- **パッケージマネージャー**：npm、yarn、またはpnpm

### 9.2 必須機能
- **レスポンシブデザイン**（モバイル、タブレット、デスクトップ対応）
- **高速なページ読み込み速度**（Next.jsの最適化機能を活用）
- **多言語対応**（必須）
  - 対応言語：日本語、英語（その他必要に応じて追加）
  - 実装方法：next-intl または next-i18next
  - 言語切り替えUI（ヘッダーに配置）
  - URL構造：`/ja/...`、`/en/...` または `/?lang=ja`、`/?lang=en`
  - 言語ごとのメタデータ・SEO対応
- **SEO対策**
  - Next.jsのメタデータAPI活用
  - 構造化データ（JSON-LD）
  - sitemap.xml自動生成（言語別）
  - robots.txt
- **アクセシビリティ対応**（WCAG 2.1 AA準拠）
- **画像最適化**（Next.js Imageコンポーネント）

### 9.3 推奨機能
- **アニメーション・スクロールエフェクト**（必須）
  - **framer-motion**（推奨）
    - React用のアニメーションライブラリ
    - カードアニメーション、スクロールアニメーションに使用
  - **react-intersection-observer**
    - スクロールトリガーアニメーション用
  - **react-spring**（代替案）
    - 物理ベースのアニメーションが必要な場合

  - react-hook-form + zod

- **Google Analytics連携**
  - next/script を活用
- **SNSシェアボタン**
- **RSSフィード**（ブログ・プレスリリース用）

### 9.4 Next.jsの特徴を活かした実装
- **多言語対応のルーティング**
  - `/[locale]/...` - 言語別ルーティング（例：`/ja/about`、`/en/about`）
  - または `next-intl` を使用した国際化ルーティング
- **動的ルーティング**
  - `/news/[slug]` - ニュース詳細
  - `/press/[slug]` - プレスリリース詳細
  - `/services/[service]` - サービス詳細
  - `/members/[slug]` - メンバー詳細（オプション）
- **API Routes**（必要に応じて）
  - `/api/contact` - お問い合わせフォーム送信
  - `/api/newsletter` - メールマガジン登録
- **静的生成（SSG）**
  - プレスリリース、ブログ記事は静的生成で高速化
- **インクリメンタル静的再生成（ISR）**
  - 定期的に更新されるコンテンツに適用
